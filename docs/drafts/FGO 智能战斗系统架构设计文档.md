## 1. 核心设计理念

### 1.1 问题的本质

FGO 战斗是一个**受约束的序列决策问题**。每回合需要在有限信息下，从众多操作组合中选出最优解。与简单游戏不同，FGO 的复杂性来自三个方向：

**资源的跨回合依赖性**：本回合选择 Arts 卡蓄 NP，影响的是下回合能否发动 NP 爆发。技能的冷却同样是跨回合资源。任何当下的操作都在消耗或积累跨回合资源。

**操作顺序的不可逆性**：技能必须在打卡之前使用，攻击强化必须在 NP 发动之前开启，否则增益完全失效。先后顺序错误，等价于浪费了一个操作机会。

**多目标的同时竞争**：同一回合内，"最大化伤害"与"蓄 NP"往往是矛盾的；"保命"与"输出"也常常冲突。系统必须有清晰的优先级体系来解决这种冲突。

### 1.2 设计的三个原则

**原则一：分层，不要用一个模块解决所有问题。**  
战略判断、战术规划、具体执行是三个不同粒度的问题，混在一起会导致逻辑混乱、难以调试。每一层只做自己粒度的决策，向下层传递结构化的指令。

**原则二：可解释，每个决策都能说清为什么。**  
"AI 选了这张卡"本身没有价值，"AI 因为当前处于蓄力阶段、这张 Arts 卡能充最多 NP 所以选它"才是有价值的决策。可解释性也便于调试和优化。

**原则三：优先级驱动，不要平权投票。**  
存活 > 输出 > 蓄力，这个优先级必须是硬性的，不能因为输出评分太高就覆盖"保命"这个约束。优先级是刚性规则，评分系统只在同一优先级内做比较。

## 2. 游戏局面的信息模型

AI 要做决策，首先要有一个完整描述"当前游戏状态"的信息模型。这个模型是整个系统的数据基础。

### 2.1 信息的四个维度

```
游戏状态
├── 我方信息
│   ├── 前排三个从者的详细状态
│   └── 后排三个从者的预备状态
├── 敌方信息
│   ├── 场上最多三个敌人的状态
│   └── 敌人的行动预告（如有）
├── 场地信息
│   ├── 当前波次 / 总波次
│   ├── 本波已过回合数
│   └── 场地 Buff（天气、地形效果等）
└── 本回合手牌
    └── 摸到的 5 张指令卡（归属从者、颜色）
```

### 2.2 从者状态的关键字段

每个从者需要追踪以下信息，这些信息直接影响决策：

|字段|用途|
|---|---|
|当前 HP / 最大 HP|判断是否濒死，是否需要保命操作|
|当前 NP 充能量（0~100%）|判断是否可以发动 NP，还差多少|
|三个技能的剩余冷却|判断哪些技能本回合可用|
|当前 Buff 列表|影响伤害估算、NP 计算|
|卡池构成（B/A/Q 各几张）|影响手牌概率与连锁规划|
|职阶 / 属性|用于克制关系计算|
|是否处于前排|只有前排从者可以操作|

**从者状态的派生指标**（由上述字段计算得出，频繁使用所以预先计算）：

- **HP 百分比**：HP / 最大 HP，用于阈值判断（如"低于 30% 视为濒死"）
- **NP 距满差值**：100% - 当前 NP，用于评估充能技能的价值
- **可用技能数**：冷却为 0 的技能数量，用于换人决策
- **有效攻击力**：基础攻击力 × 当前攻击 Buff 乘区，用于伤害估算

### 2.3 敌人状态的关键字段

|字段|用途|
|---|---|
|当前 HP / 最大 HP|判断是否可以本回合秒杀|
|破防条列表（各条 HP）|规划打穿时机|
|当前 Buff / Debuff|估算实际受到的伤害|
|行动预告类型|预判是否本回合放 NP，提前部署防御|
|NP 充能条进度|推算敌人几回合后放 NP|
|职阶 / 属性|克制关系计算|
|破防条触发效果|决定是否主动在本回合打穿|

### 2.4 手牌信息

5 张手牌每张需要知道：

- 归属于哪个前排从者（索引 0/1/2）
- 卡的颜色（Buster / Arts / Quick）
- 该卡的段数（影响 NP 充能和星生成计算）

## 3. 整体架构：三层决策模型

### 3.1 架构图

```
┌──────────────────────────────────────────────────────┐
│                    每回合入口                         │
│              读取最新游戏状态                          │
└──────────────────────────┬───────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────┐
│                  第一层：战略层                        │
│                                                      │
│   输入：游戏状态（全局）                               │
│   处理：识别当前局面所处的战略阶段                     │
│   输出：战略阶段标签 + 阶段紧迫程度                    │
│                                                      │
│   例：「当前处于蓄力阶段，紧迫程度：普通」              │
└──────────────────────────┬───────────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────┐
│                  第二层：战术层                        │
│                                                      │
│   输入：战略阶段标签 + 游戏状态                        │
│   处理：将战略目标翻译成本回合具体战术目标              │
│   输出：本回合目标列表（主目标 + 次目标 + 约束条件）    │
│                                                      │
│   例：主目标「充 NP」，次目标「清掉左边残血小兵」       │
│       约束「不得让 2 号从者死亡」                      │
└──────────────────────────┬───────────────────────────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
       ┌──────────┐ ┌──────────┐ ┌──────────┐
       │ 换人决策 │ │ 技能决策 │ │ 选卡决策 │
       │  子系统  │ │  子系统  │ │  子系统  │
       └──────────┘ └──────────┘ └──────────┘
              │            │            │
              └────────────┼────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────────┐
│                   操作执行序列                         │
│   按固定顺序输出：换人 → 御主技能 → 从者技能 → 打卡    │
└──────────────────────────────────────────────────────┘
```

### 3.2 三层的职责边界

|层级|职责|不负责|
|---|---|---|
|战略层|判断"现在整体局势是什么"|不决定具体操作|
|战术层|判断"本回合要达成什么目标"|不决定用哪个技能、选哪张卡|
|执行层|判断"具体怎么操作才能达成目标"|不判断局势好坏|

这种边界确保了：修改战略判断逻辑，不会影响技能决策逻辑；更换选卡算法，不会影响战略阶段的识别。

## 4. 第一层：战略层 —— 判断局面阶段

### 4.1 战略阶段的定义

战略阶段是对当前局面的"一句话总结"，决定了本回合一切行为的基调。共分五个阶段：

**防御阶段（DEFENSE）**

> 当前局面存在生存威胁，首要任务是保住从者不死。

触发条件：

- 前排有从者 HP 低于 30% 且敌人下回合放 NP
- 前排有 2 个或以上从者处于濒死状态
- 敌人本回合有必中全体攻击且己方没有无敌/回避

---

**BOSS 阶段（BOSS）**

> 当前面对的敌人有破防条，或处于最终波次的 BOSS 战。

触发条件：

- 当前是最终波次（wave == totalWave）
- 场上有敌人拥有多段 HP 破防条

---

**爆发阶段（BURST）**

> 当前有充足的输出资源（NP、攻击 Buff），适合最大化伤害。

触发条件：

- 前排有 2 个以上从者 NP 达到 100%
- 前排主力输出从者 NP 达到 100% 且场上有关键攻击 Buff 技能可用

---

**清场阶段（CLEAR）**

> 敌人整体 HP 很低，本回合输出即可清场，不需要浪费大量资源。

触发条件：

- 所有活着的敌人的 HP 总和低于本回合最低预计伤害的 80%

---

**蓄力阶段（CHARGE）**

> 以上条件均不满足时的默认状态，重心是积累资源为下次爆发做准备。

触发条件：无，兜底阶段

### 4.2 阶段判断的优先级

多个阶段条件可能同时满足，判断有严格的优先级顺序：

```
防御阶段  >  BOSS阶段  >  爆发阶段  >  清场阶段  >  蓄力阶段
  （最高）                                          （最低/默认）
```

防御阶段是唯一不可被其他阶段覆盖的情况——即便 NP 已满、即便是 BOSS 战，生存威胁必须先处理。

### 4.3 阶段的附属属性：紧迫程度

同一战略阶段内，还需要区分紧迫程度，它影响战术层的激进程度：

|紧迫程度|含义|典型场景|
|---|---|---|
|危急（Critical）|必须本回合解决，否则可能灭队|敌人下回合放 NP 且有人濒死|
|高（High）|优先处理，有余地但不多|敌人 NP 充能即将满|
|普通（Normal）|按计划推进|正常蓄力中|

## 5. 第二层：战术层 —— 确定本回合目标

### 5.1 战术目标的结构

战术层的输出不是一个单一目标，而是一个有层次的目标包：

```
本回合目标包
├── 主目标（Primary Goal）：必须优先完成的核心目标，只有一个
├── 次目标列表（Secondary Goals）：在不影响主目标的前提下顺带完成
└── 约束条件（Constraints）：绝对不能违反的底线（如"必须保住某从者"）
```

### 5.2 各战略阶段的战术目标映射

**防御阶段 → 战术目标**

- 主目标：让濒死的从者撑过本回合
- 次目标：降低敌方威胁（如打断 NP 充能、减少敌人数量）
- 约束：不得触发任何可能导致死亡的高风险操作（如强行爆发导致敌人 Rage 反击）

---

**BOSS 阶段 → 战术目标**

需要先判断破防条状态再定目标：

- 若本回合能打穿破防条且打穿后效果有利 → 主目标：**打穿破防条**
- 若本回合不能打穿 → 主目标：**最大化对当前 HP Bar 的伤害**，避免伤害溢出浪费
- 若 BOSS 处于最后 HP Bar → 主目标：**歼灭**，全力输出

---

**爆发阶段 → 战术目标**

- 主目标：最大化本回合总伤害
- 次目标：触发 NP，消耗暴击星
- 约束：攻击 Buff 技能必须在 NP 发动之前使用

---

**清场阶段 → 战术目标**

- 主目标：以最小资源消耗清掉所有敌人
- 次目标：顺带蓄 NP（为下波做准备）
- 约束：不浪费高冷却技能在已经死透的局面

---

**蓄力阶段 → 战术目标**

- 主目标：最大化本回合 NP 充能量
- 次目标：生成暴击星（为下次爆发蓄星），快速清掉最弱的小兵
- 约束：不使用会消耗 NP 的技能（除非有别的原因）

### 5.3 目标的优先攻击目标选择

战术层还需要指定本回合优先攻击哪个敌人，选择逻辑如下：

|情景|优先攻击目标|
|---|---|
|清场阶段|HP 最低的敌人（最容易秒杀）|
|爆发阶段|HP 最高 / 威胁最大的敌人（避免浪费在已濒死的目标上）|
|BOSS 阶段|固定为 BOSS（通常场上只有 BOSS）|
|防御阶段|NP 充能最高的敌人（优先击杀或打断）|
|蓄力阶段|与己方阵容克制关系最有利的敌人|

## 6. 第三层：执行层 —— 具体操作决策

执行层接收战术目标，转化为实际的操作序列。操作序列有固定的执行顺序：

```
① 换人（Order Change）  ——  改变场上人员构成
② 御主技能              ——  全局增益/工具性效果
③ 从者技能              ——  从者专属增益/控制
④ 指令卡（含 NP）       ——  主要伤害/充能来源
```

这个顺序不可改变，因为每一步都可能影响后续步骤的价值：

- 换人后，场上从者组成变了，技能和卡的可选项都变了
- 技能开完之后，卡的实际价值才能准确估算（Buff 已激活）
- NP 必须在技能 Buff 生效之后再发动，才能拿到加成

## 7. 技能决策子系统

### 7.1 技能决策的两个问题

技能决策解决两个独立的问题：

1. **用哪些技能**：本回合应该触发哪些技能（可能触发 0~9 个）
2. **什么顺序用**：多个技能的使用顺序（顺序直接影响效果）

### 7.2 技能的分类与决策规则

将游戏中所有技能效果归为以下几类，每类有对应的触发逻辑：

---

**A 类：紧急保命类**（无敌、回避、复活）

决策规则：

- 触发条件极为严格，不在紧急情况下绝对不用（留着才有价值）
- 触发条件：敌人下回合放 NP **且**己方该从者会被一击致死
- 或：该从者当前 HP 低于 20% 且没有其他救援手段

不触发条件：

- 敌人是普通攻击（可以承受）
- 己方有治疗或减伤可以替代
- 该从者已有 Guts 效果（自带复活）

---

**B 类：控制敌人类**（晕眩、魅惑、封印、沉默）

决策规则：

- 最高价值时机：敌人 NP 充能即将满，用眩晕打断
- 次优时机：BOSS 阶段，削弱 BOSS 的行动能力
- 低价值时机：普通蓄力阶段的小兵（浪费高价值技能）

---

**C 类：攻击强化类**（攻击 UP、卡色强化、暴击伤害 UP、NP 威力 UP）

决策规则：

- 只在"有配套的伤害输出"时使用，否则 Buff 会在下回合自然消失
- 配套 NP：主力输出从者 NP 满了 → 开攻击 Buff → 发动 NP（三步顺序）
- 配套暴击：场上暴击星超过 20 颗 → 开暴击伤害 Buff → 优先打 Quick 链集星+暴击
- 单独开攻击 Buff（没有配套）：只在爆发阶段或 BOSS 阶段可接受

---

**D 类：NP 充能类**（直接充 NP 的技能）

决策规则：

- 核心判断：用完之后能否达到 100%？
    - 能达到 → 强烈建议本回合使用（配合 NP 发动）
    - 不能达到但能达到 70% 以上 → 可以使用（更快蓄满）
    - 用完仍不足 50% → 暂缓，等 NP 再高一些再用（此时用收益最低）

---

**E 类：治疗/回复类**

决策规则：

- 不要提前用，要在从者 HP 低于特定阈值时才用（阈值建议 50%）
- 濒死（低于 30%）时无条件触发
- 治疗量有限时，优先治疗主力输出从者

---

**F 类：生星/聚星类**

决策规则：

- 聚星技能（让某从者优先获得暴击星）：在暴击星充足（20 颗以上）时才有价值
- 生星技能：在蓄力阶段且下回合计划打暴击时使用

### 7.3 技能使用顺序规则

顺序错误是最常见的操作失误，必须明确规定：

```
正确顺序：
  NP 充能技能  →  攻击强化技能  →  打出指令卡（含 NP）

错误示范：
  先放 NP  →  再开攻击 Buff   ← NP 伤害已经打出去了，Buff 没有生效
  先开 NP 充能  →  NP 还没满  → 攻击强化  → 发现不能放 NP（卡顺序逻辑错误）
```

技能使用顺序优先级表（数字越大越先执行）：

|优先级|技能类型|原因|
|---|---|---|
|5（最先）|敌人控制（眩晕）|需要先确认控制生效，再规划后续|
|4|NP 充能技能|先充 NP，确认 NP 够用再开 Buff|
|3|攻击强化 / NP 威力 UP|Buff 在打卡前生效|
|2|卡色强化 / 暴击强化|同上|
|1（最后）|保命技能（无敌/回避）、治疗|最后使用，贴近受伤害时刻|

### 7.4 技能的目标选择

部分技能有目标（对友方单体增益、对敌人单体控制）：

**友方目标选择原则**：

- 攻击强化 → 优先给 NP 满了的主力输出从者
- 治疗 → 优先给 HP 最低的前排从者
- NP 充能 → 优先给离 100% 最近的从者（最少浪费）
- 聚星 → 优先给本回合会暴击的从者

**敌方目标选择原则**：

- 眩晕/控制 → 优先给 NP 充能最高的敌人
- 减防 → 优先给 HP 最多的敌人（最大化收益）

## 8. 指令卡决策子系统

### 8.1 指令卡决策的完整流程

```
第一步：给 5 张手牌分别打分（单张卡价值评估）
       ↓
第二步：枚举所有合法的 3 卡顺序组合（最多 60 种）
       ↓
第三步：给每种组合计算总得分（含连锁加成）
       ↓
第四步：判断是否有 NP 要发动，并决定 NP 插入位置
       ↓
第五步：输出最终操作序列（含 NP 的最终 3 步）
```

### 8.2 单张指令卡的价值维度

每张指令卡的价值由三个维度合成：

**伤害价值**：这张卡实际打出的伤害估算值

- 受卡色（B > A > Q 基础伤害）影响
- 受从者当前攻击 Buff 影响
- 受职阶克制影响

**NP 充能价值**：打出这张卡能给所属从者充多少 NP

- Arts 卡充能最多，Quick 次之，Buster 最少
- 受从者自身 NP 获取率影响
- 从 NP 不足 100% 的从者打 A 卡，充能价值更高

**暴击星价值**：打出这张卡能生成多少暴击星

- Quick 卡生星最多
- 受从者星生成率影响
- 场上暴击星已经很多时，额外生星的边际价值下降

### 8.3 价值权重随战略阶段动态变化

三个维度的权重不是固定的，由当前战略阶段决定：

|战略阶段|伤害权重|NP 充能权重|暴击星权重|
|---|---|---|---|
|爆发阶段|★★★★★|★★☆☆☆|★★★☆☆|
|蓄力阶段|★★☆☆☆|★★★★★|★★★☆☆|
|防御阶段|★☆☆☆☆|★★★☆☆|★☆☆☆☆|
|清场阶段|★★★★☆|★★☆☆☆|★★☆☆☆|
|BOSS 阶段|★★★★★|★★★☆☆|★★★☆☆|

**这个权重表意味着**：同样一张 Arts 卡，在蓄力阶段的价值评分远高于爆发阶段；同样一张 Buster 卡，在爆发阶段的评分远高于蓄力阶段。

### 8.4 首卡加成机制

每回合打出的第一张卡会给本回合**所有指令卡**带来加成，因此首卡的选择有特殊权重：

|首卡类型|加成效果|最适合的阶段|
|---|---|---|
|Buster 首卡|本回合全体攻击力 +50%|爆发阶段、BOSS 阶段|
|Arts 首卡|本回合全体额外 NP 充能 +20%|蓄力阶段|
|Quick 首卡|本回合额外生星 +20%|蓄力/蓄星阶段|

首卡选择的决策规则：按照当前阶段的主目标，匹配对应的首卡类型。

### 8.5 同色连锁加成（Card Chain）

当 3 张卡颜色相同时，触发连锁加成：

**Buster 链（BBB）**

- 效果：第一张 Buster 卡额外打出一次全体攻击（相当于首卡伤害 +50%）
- 最适合：爆发阶段、BOSS 阶段
- 代价：通常牺牲了 Arts/Quick 卡的充能/生星价值

**Arts 链（AAA）**

- 效果：每张 Arts 卡额外充 NP +20%（三张合计 +60%）
- 最适合：蓄力阶段、NP 快满但还差一口气的情况
- 代价：Arts 卡伤害低，伤害产出损失大

**Quick 链（QQQ）**

- 效果：额外生成 +10 颗暴击星
- 最适合：蓄星阶段
- 代价：伤害和 NP 充能均不理想，通常是收益最低的链

**如何在组合评分中体现链式加成**：在枚举所有 60 种排列时，检测是否触发连锁，触发则在该组合的总分上叠加连锁奖励分（按当前阶段权重折算）。

### 8.6 勇者链（Brave Chain）

当 3 张卡全部属于同一个从者时，触发勇者链：

- 效果：在第 3 张卡之后额外追加一次 Extra 攻击（该从者的 Extra 卡）
- Extra 攻击通常为全体攻击，伤害可观
- 勇者链的价值：约等于多打一张高倍率卡

勇者链与同色链可以同时触发，例如一个从者的三张 Buster 卡组成"BBB 勇者链"，同时享受两种加成。

### 8.7 NP 发动的位置选择

NP 不占用手牌的一张，但属于本回合操作序列中的"一步"。当有从者 NP 满了，需要决定将 NP 插入到哪个位置：

**位置 1（NP 作为首卡）**

- 好处：享受 NP 卡类型的首卡加成（如 Buster NP 可触发 Buster 首卡全体攻击加成）
- 坏处：NP 打出后 NP 清零，后续普通卡的充能效率不受 NP 本身影响

**位置 2（NP 作为第二步）**

- 好处：先打一张普通卡蓄点 NP（在 NP 溢出时会有过充，部分机制有用）
- 坏处：失去首卡加成

**位置 3（NP 作为最后一步）**

- 好处：前两张普通卡已经打出，可以充分利用前两张卡的 NP/星收益
- 坏处：失去首卡加成，且 NP 是全回合最后打，之前的卡不能受益于 NP 后的效果

**决策建议**：

- 主力 DPS 的 NP 且 NP 卡是 Buster 类型 → 放位置 1，利用 Buster 首卡加成
- 辅助性 NP（有回 NP 或星的效果）→ 放位置 1 或 2
- 多个 NP 同时发动时 → 先放辅助 NP，再放输出 NP（让增益先生效）

## 9. 换人决策子系统

### 9.1 换人的前提条件

换人消耗玛修的 Order Change 御主技能槽（或专属换人技能），每场战斗次数有限。因此换人必须有明确的收益，不能随意触发。

前提：

- Order Change 技能可用（未使用过或已冷却）
- 后排有存活的从者
- 换人带来的收益超过"留着技能备用"的机会成本

### 9.2 触发换人的收益评估

以下场景按收益从高到低排列：

**最高收益：后排 NP 满，前排 NP 未满且本回合是爆发节点**

- 将后排已蓄满 NP 的从者换入，立即参与本回合爆发输出
- 尤其适合多 NP 同时爆发的策略

**高收益：前排某人即将死亡，且其技能已全冷却**

- 该从者战斗力为零，换进后排存活但未上场的从者，立即获得可用技能

**高收益：BOSS 战前，主力输出从者不在前排**

- 在进入最终 BOSS 波之前的最后机会换入主力，确保输出爆发时人员到位

**中等收益：前排技能全 CD，后排有多个立即可用的关键技能**

- 当前排从者能做的事情很少时，换人相当于直接解锁了新的操作空间

**低收益 / 不建议换人的情况**：

- 前排某从者技能冷却只剩 1 回合（等一下就好了，不需要换）
- 后排从者没有比前排明显更高的立即战斗价值
- 当前处于防御阶段（换人会消耗操作机会，而当前最需要的是快速执行保命操作）

### 9.3 换人的目标选择

**换出谁**（从前排移除的从者）：

- 战斗贡献最低的从者：HP 很低且技能全 CD，或主要职能已经完成（如辅助技能已经全开过了）

**换入谁**（从后排进入前排的从者）：

- NP 已满且尚未发动的从者（立即输出价值）
- 拥有当前最需要效果的技能（如当前需要控制，后排有眩晕技能）
- 与当前场上职阶克制敌人的从者

## 10. 御主技能子系统

### 10.1 御主技能的特点

御主技能（3 个）不属于任何从者，全局共享冷却，且通常有相当长的冷却时间（约 15 回合）。因此每次使用都需要确保物尽其用。

### 10.2 常见御主技能类型与决策

**攻击强化型**（如灰大叔 A 技能）：给全体前排从者增加攻击

- 决策：与从者攻击技能相同，配合 NP 或大量暴击卡时使用
- 不建议在蓄力阶段单独使用

**NP 充能型**（如灰大叔 B 技能）：给指定从者充 NP

- 决策：目标从者 NP 在 60% 以上时使用效率最高
- 若目标从者 NP 已满，改为充第二优先的从者

**治疗型**：回复前排或指定从者的 HP

- 决策：有人 HP 低于 40% 时触发，低于 20% 时无条件触发

**Order Change 型**：见第 9 节

### 10.3 御主技能与从者技能的顺序

御主技能与从者技能同属于"打卡之前"的操作窗口，顺序上：

- 御主充能技能 → 从者攻击 Buff → 再打卡（NP 此时已满）
- 御主治疗技能：可以在从者技能之后使用，不影响攻击顺序

## 11. Buff 状态管理

### 11.1 Buff 系统的作用

Buff（增益/减益效果）渗透在整个决策体系中：

- 伤害估算需要考虑攻击 UP Buff
- 技能决策需要知道哪些 Buff 已经存在（避免重复叠加无意义的同类 Buff）
- NP 充能计算需要考虑 NP 获取 UP Buff
- 暴击星计算需要考虑星生成 UP Buff

### 11.2 Buff 的核心属性

每个 Buff 需要追踪：

|属性|说明|
|---|---|
|效果类型|攻击 UP、防御 DOWN、NP 充能 UP 等|
|效果数值|如"攻击 +30%"中的 30%|
|剩余回合数|还有几回合消失（-1 表示永久）|
|来源|哪个技能/效果给的（用于可解释性）|
|是否可叠加|同类 Buff 是否可以叠加（大多数 FGO Buff 可以叠加）|

### 11.3 Buff 对决策的影响

**对技能决策的影响**：

- 若攻击 UP Buff 剩余回合数大于 0，不需要重复开同类技能（除非叠加收益够高）
- 若无敌 Buff 已经在身上，不需要再开另一个无敌技能

**对选卡决策的影响**：

- 攻击 UP 存在时，Buster 卡的伤害价值更高（乘区叠加）
- NP 获取 UP 存在时，Arts 卡的 NP 充能价值更高
- 星生成 UP 存在时，Quick 卡的生星价值更高

## 12. 伤害与收益估算

### 12.1 估算而非精确计算

精确的 FGO 伤害公式极为复杂（含随机浮动、多个独立乘区、段数计算等），在 AI 决策中不需要精确值，只需要**能比较大小的相对估算**。

估算公式的简化原则：

- 忽略随机浮动（使用期望值）
- 忽略次要修正（如攻防等级差）
- 保留关键乘区：卡色系数、攻击 Buff、职阶克制

### 12.2 估算结果的三个用途

**用途一：判断是否能本回合秒杀目标**

- 估算本回合所有卡打完后的最低总伤害（保守估计）
- 与目标当前 HP 比较，低于目标 HP × 0.8 则认为不能稳定秒杀

**用途二：比较不同卡组合的相对输出**

- 不需要精确值，只需要排序，找出伤害期望最高的组合

**用途三：判断 NP 能否打穿破防条**

- 估算主力从者 NP 的伤害期望
- 与破防条当前 HP 比较，判断是否需要额外的减防 Buff 辅助

### 12.3 连锁加成的估算方法

连锁加成不能忽略，否则会低估同色链的价值：

- Buster 链的额外价值 ≈ 第一张 Buster 卡伤害的 50%
- Arts 链的额外价值 ≈ 三张 Arts 卡各 +20% NP，折算为"提前几步到达 100%"的回合价值
- Quick 链的额外价值 ≈ 额外 10 颗暴击星，折算为下回合暴击伤害期望的增量

## 13. 特殊机制处理

### 13.1 破防条（多段 HP Bar）

破防条是 FGO BOSS 战中最独特的机制，有几个关键处理点：

**伤害不溢出规则**：打穿当前 HP Bar 时，超出的伤害不会计入下一条（直接归零进入下一条）。这意味着：

- 若本回合不能打穿，要避免"刚好差一点打穿"的情况，因为下回合 BOSS 会满血开始下一条
- 若本回合能轻松打穿，要评估是否值得现在打穿

**破防触发效果评估**：

- 打穿某些 BOSS 的破防条会触发反制效果（如全体攻击、全员 Buff 清除）
- 若触发效果危险，需要确保己方有防御技能就绪，才主动打穿
- 若触发效果对 BOSS 有害（如 BOSS 自我施加弱体），则积极打穿

**多破防条的规划**：

- BOSS 有 3 条 HP Bar 时，要规划好"第几回合打穿第一条，第几回合打穿第二条"
- 通常在 NP 就绪的回合打穿，避免在 NP 冷却期内陷入高难度破防条阶段

### 13.2 Guts（不死）效果

Guts 在 HP 归零时触发，让从者以 1 点 HP 存活。在 AI 决策中的处理：

- 拥有 Guts 的从者不视为"需要立即救援"，即便 HP 极低
- Guts 不是无敌——仍然会受到伤害，只是死亡时触发一次，需要区分
- 建议：Guts 存在时，该从者的濒死阈值可以放宽（不需要立即治疗）

### 13.3 即死与高即死率 Debuff

某些敌人技能给从者施加高即死率 Debuff，命中时概率直接死亡：

- AI 需要识别"该从者身上有高即死率 Debuff"
- 处理策略：优先为该从者激活无敌/回避，或通过换人让其退出前排

### 13.4 Sleep（睡眠）效果

睡眠与晕眩不同——睡眠会在受到伤害时强制解除，且解除时施加额外效果（有时对 BOSS 有利）：

- 对 BOSS 使用睡眠技能时，下一步的指令卡选择需要考虑"是否让 BOSS 在当回合醒来"
- 若睡眠触发的觉醒效果对己方不利，则考虑本回合不攻击该 BOSS（仅攻击其他目标）

```
       │          ┌───────────────────┘
       │          │   （技能生效后更新状态的 Buff 部分）
       │          ▼
       │   ┌─────────────────────┐
       │   │  选卡子系统          │
       │   │  输入：更新后的状态   │
       │   │       + 目标包       │
       │   │  → 最优卡组合        │
       │   │  → NP 插入位置       │
       │   └──────────┬──────────┘
       │              │
       └──────┬───────┘
              │
              ▼
        操作执行序列
        （有序列表，供外部执行）
```

### 14.2 状态更新时机

在单回合内，状态需要在特定时刻更新（模拟 Buff 生效）：

1. 换人决定后：更新前排/后排人员构成
2. 每个技能使用后：将技能效果（Buff）更新到对应从者的 Buff 列表
3. 选卡打分时：使用"已含当前所有技能 Buff"的状态进行估算

## 15. 系统扩展设计

### 15.1 新机制的接入方式

FGO 会不断引入新的游戏机制（新职阶、新 Buff 类型、特殊副本规则等）。系统应设计为插件式扩展：

**新 Buff 类型**：在 Buff 分类中增加新条目，并在对应的估算函数中添加该 Buff 的影响计算。无需修改核心决策框架。

**新从者机制**（如特殊 NP 充能方式、额外行动次数）：在从者状态模型中增加字段，在技能/选卡子系统中增加对应的处理规则。

**特殊副本规则**（如"每回合结束时全体扣 HP""特定从者禁止出场"）：在战略层添加"特殊规则感知"模块，将这些约束注入到目标包的约束条件列表中。

### 15.2 阵容策略感知

不同的阵容组合有截然不同的最优战法，系统可引入"阵容模式识别"：

|阵容模式|识别特征|对应策略调整|
|---|---|---|
|Arts 循环队|前排大量 Arts 卡 + 多个 NP 获取 UP 技能|强化 Arts 链的权重，以连续发动 NP 为核心目标|
|Buster 暴力队|Merlin 在队、大量 Buster 卡 + 星获取|攒星优先，每回合 Buster 链，暴击爆发|
|Quick 循环队|大量 Quick 卡 + 星获取 UP + 暴击伤害 UP|Quick 链生星，持续暴击输出|
|奶妈辅助队|多个治疗/无敌/充能辅助从者|优先保命，以续航为核心，等主力蓄满|

阵容模式识别后，对选卡权重表和技能触发阈值做整体调整，无需逐条修改规则。

### 15.3 高难度副本的特化支持

常规策略适合大部分副本，但高难本（如 CQ 挑战、终局等）需要特化：

**多回合规划**：在高难副本中，AI 可以在回合开始时向前预测 2~3 回合，评估"现在蓄力 → 下回合爆发 vs. 现在爆发 → 下回合资源空虚"哪个更优。

**固定路线执行**：部分高难副本有唯一正确解（特定回合必须放 NP、特定回合必须有无敌），可以为这些副本预设固定操作脚本，AI 识别到对应副本后切换至脚本执行模式。

## 16. 分阶段落地路线

### 16.1 四个迭代阶段

**第一阶段：基础可用（MVP）**

目标：比随机选卡明显更好，能通过大部分普通副本

实现内容：

- 只区分"爆发"和"蓄力"两个战略阶段
- NP 满了就用（配合攻击 Buff）
- 有人濒死就触发保命技能
- 按阶段优先 Arts 链（蓄力）或 Buster 链（爆发）

验证指标：通过率比随机选卡提升 40% 以上

---

**第二阶段：战术完整**

目标：覆盖 5 个战略阶段，技能顺序正确，选卡评分可用

实现内容：

- 5 阶段战略判断
- 完整的技能分类决策（含使用顺序）
- 60 种卡组合枚举与链式加成计算
- 基础换人决策

验证指标：能稳定通过大部分活动关卡，高难本有一定成功率

---

**第三阶段：机制覆盖**

目标：处理破防条、特殊 Buff、BOSS 机制等复杂场景

实现内容：

- 破防条分析与打穿时机决策
- Sleep/Guts/即死等特殊机制处理
- 阵容模式感知与权重自动调整
- 御主技能联动完善

验证指标：能稳定通过高难副本，CQ 胜率显著提升

---

**第四阶段：优化与调参**

目标：系统稳定，参数调优，支持扩展

实现内容：

- 收集战斗日志，分析决策质量
- 调整权重参数（哪些阶段的权重设定不合理）
- 新机制的插件式接入流程标准化
- 多回合预测（针对特定高难副本）

### 16.2 各模块的依赖关系

```
战略层（阶段判断）
    ↓ 依赖
战术层（目标生成）
    ↓ 依赖
技能子系统  ←──  Buff 状态管理
    ↓
选卡子系统  ←──  伤害估算模块
    ↓
换人子系统  ←──  从者价值评估

伤害估算模块  ←──  克制关系数据
Buff 状态管理  ←──  技能效果数据库
```

各模块的开发和测试可以独立进行，建议开发顺序：

```
伤害估算  →  Buff 管理  →  战略层  →  战术层
→  技能子系统  →  选卡子系统  →  换人子系统
→  联调  →  特殊机制  →  调参
```
